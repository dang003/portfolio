CARGO

Case Study

CARGO gives travelers a platform where they can store, organize, and access their files. Photos, plans, and notes can all be created or uploaded with this cloud storage site.

problem.

Many cloud storage site users are frustrated by the difficulty of organizing their files. Users feel the current organization options aren’t accessible enough.

solution.

User files are stored in trips. Users also have the ability to add tags to their files, which lets them customize their organization. Users will be able to access files by opening a trip or directly searching by title or tag.

research.

Survey Results

88% of users go to cloud storage for personal reasons, like storing family photos.

75% mentioned organization as a major frustration with their current services.

Competitive Analysis

  • I analyzed 3 major cloud storage sites: Google Drive, Dropbox, and Evernote.

  • I decided that it would benefit us more if our site targets a niche market.

information architecture.

User Stories

I ended up with four high priority user stories, based on my survey results from earlier.

I want to make an account High
I want to create a new trip High
I want to view a picture High
I want to search my trips High
User Flows

I created user flows for high priority stories. The user flows were laid out in a way that allows the user to take multiple directions to arrive at the end goal.

View User Flows
Wireframes

Next, I did some testing with basic wireframes. I gave each user 3 basic tasks:

  • Sign in to an account

  • Upload pictures

  • Tag a note

These 3 tasks all use different features of the navigation, so I was able to understand how effective the site worked as a whole.

branding.

Logo

  • I turned the "O" in cargo into the mark to associate the brand with the name.

  • I designed this logo to look simple, friendly, and secure.

  • I made sure to match the mark to the width of the font, Roboto Bold, to make it look more natural.

Moodboard

  • I wanted this moodboard to communicate excitement, adventure, and simplicity.

  • Each color here represents something different about the brand: Red = Excitement, Blue = Trust, Orange = Friendly, and Yellow = Happiness.

Icons

  • I wanted the icons to look fun, while still being clear to the user what each one means.

  • To connect these icons, I added a touch of orange (the main brand color) to each one, except search. I wanted the search icon to remain clear to all users.

testing.

Preference Testing

I created a few versions of important features' design, and held preference tests to see how users feel about them. Which do you like better?

The results were split, and users did not seem happy with either. I realized this wasn't solving user problems, so I took their feedback and created a new design.

The goal behind this was to personalize a user's experience. This new design looks clean and provides the user with helpful information.

Usability Testing

I returned to user testing and gave users three tasks to complete on a prototype of my app. The tasks were:

  • Make an account

  • Upload an image

  • Tag a note

Final Changes

From my testing, I created some solutions to enhance user experience. Some of my final changes were:

  • fixing the upcoming trip notification

  • organizing all navigation to the left side of the screen

  • consolidating the search function to an icon and a screen overlay

  • using icons on the cards so users can quickly identify what they’re looking at


I’m happy with this end product, as it solves problems users commonly face and reflects the core ideas of the brand image.

conclusion.

CARGO was built from research and refined by user testing. I’m thankful for all the ideas that didn’t work out, because they let me build an even better app. I experienced the full design process of a project, from research to finished product. I learned that, as a designer, I should never assume I know what the users want.


In the future, with a full scope of the project in mind, I will be able to create better research questions and tests to help me arrive at a polished final design that will address the user’s needs.
